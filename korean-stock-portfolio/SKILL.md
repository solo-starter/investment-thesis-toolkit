# 한국 주식 포트폴리오 분석 스킬

분기별 한국 주식 포트폴리오 갱신, 적정가 재산정, 캘린더 등록을 일관된 방법론으로 수행하는 스킬이다.

## 언제 트리거하는가

### 자연어 트리거
- "포트폴리오 업데이트해줘", "분기 실적 반영해줘"
- "한국 주식 적정가 분석"
- 기존 `한국주식_포트폴리오.xlsx` 파일을 첨부하며 "갱신해줘", "다시 분석해줘"
- "(종목명) 포트폴리오에 추가/제거"

### 슬래시 명령어 트리거 (Cowork에서 사용)
스킬은 다음 5개의 슬래시 명령어를 인식한다. 각 명령어의 상세 워크플로우는 `commands/` 디렉토리의 해당 파일을 참조한다.

| 명령어 | 영문 별칭 | 용도 | 상세 파일 |
|--------|----------|------|----------|
| `/적정가산정 [종목]` | `/fairvalue` | 종합 적정가 평가 (4단계 시나리오) | `commands/적정가산정.md` |
| `/실적분석 [종목] [분기]` | `/earnings` | 분기 실적 반영, 적정가 재산정, 히스토리 추가 | `commands/실적분석.md` |
| `/실적프리뷰 [종목]` | `/earnings-preview` | 차기 실적 발표 전 시나리오 분석 | `commands/실적프리뷰.md` |
| `/논리점검 [종목]` | `/thesis` | 분기별 히스토리로 투자 논리 추적 | `commands/논리점검.md` |
| `/ETF분석 [ETF명]` | `/etf` | 편입 종목 가중평균으로 ETF 적정가 산출 | `commands/ETF분석.md` |

**중요**: 사용자가 위 슬래시 명령어를 입력하거나 자연어로 동등한 표현을 사용하면, 즉시 해당 `commands/[명령어].md` 파일을 읽고 그 워크플로우를 따른다.

## 핵심 원칙

1. **데이터는 항상 새로 수집**: 주가, 시가총액, 컨센서스는 매 실행 시 웹 검색으로 최신값을 가져온다. 이전 분석의 숫자를 재사용하지 않는다.
2. **방법론은 일관 유지**: 시나리오 정의, PER 밴드 적용, 종목별 산정 방식은 이 스킬의 가이드를 따른다.
3. **덮어쓰기 + 히스토리**: 메인 5개 시트는 최신 상태로 갱신하고, "분기별 히스토리" 시트는 한 줄씩 누적한다.
4. **추정의 한계 명시**: 모든 적정가는 가정의 함수라는 점, 컨센서스 목표가와 교차 검증할 것을 산출물에 항상 포함한다.

## 작업 흐름 (포트폴리오 종합 갱신 시)

개별 슬래시 명령어가 아닌 "포트폴리오 업데이트" 같은 종합 요청 시 따르는 흐름이다.

### Step 1: 작업 범위 확인
사용자에게 다음을 확인한다 (생략 가능 - 명확한 경우 진행):
- 종목 리스트 변경 여부 (추가/제거)
- 기존 엑셀 파일 첨부 여부 (있으면 히스토리 보존, 없으면 신규 생성)
- 캘린더 등록 여부 (기본: 등록함)

### Step 2: 종목 데이터 수집
각 종목에 대해 다음을 검색한다. 검색 쿼리 템플릿은 `references/data_collection.md` 참조.

**필수 수집 항목** (종목당):
- 현재가 (KRW), 시가총액, 52주 최고/최저
- TTM EPS (가능한 경우), 배당수익률
- 12개월 컨센서스 목표주가 평균, 매수/매도 애널리스트 수
- 차기 실적 발표 예정일
- 2026/2027 영업이익·순이익 컨센서스 (적정가 산정용)
- 12M Forward PER (있는 경우)

### Step 3: 적정가 산정
`references/valuation_method.md`의 방법론 적용.

**종목 분류별 산정 방식**:

| 분류 | 종목 예시 | 산정 방식 |
|------|----------|----------|
| 일반 흑자 종목 | 삼성전자, SK하이닉스, 효성중공업 등 | PER × 12M Forward EPS |
| 지주사 | SK스퀘어, LS | NAV × (1 − 할인율) |
| 적자/소이익 성장주 | 두산로보틱스, 레인보우로보틱스 | PSR × 매출 |

**4단계 시나리오** (모든 종목 공통):
1. **보수**: 사이클 정점 우려 또는 PER 밴드 하단
2. **기준**: 컨센서스 평균 Target PER (밴드 중위)
3. **낙관**: 사이클 강세 지속 가정 (밴드 상단)
4. **AI 재평가**: 산업 구조가 재정의될 경우 글로벌 Peer 평균 또는 그 이상

시나리오별 PER 밴드 기본값은 `references/per_bands.md` 참조.

### Step 4: 엑셀 파일 생성
6개 시트를 작성한다:

1. **포트폴리오 종합** — 11개 종목 × 12개 항목
2. **실적 발표 일정** — 차기 발표일 정리
3. **재무지표 상세** — PER/PBR/배당/목표주가/투자의견
4. **섹터 분석** — 섹터별 투자 포인트 코멘트
5. **적정가 분석** — 4단계 시나리오 + 컨센서스 비교
6. **분기별 히스토리** — 매분기 한 줄씩 누적

**시각적 일관성**:
- 섹터 색상: 반도체(파랑 E7F3FF), AI 전력 인프라(노랑 FFF2CC), 피지컬 AI 로봇(초록 E2EFDA)
- 시나리오 색상: 보수(빨강 FCE4D6), 기준(파랑 DDEBF7), 낙관(초록 E2EFDA), 재평가(노랑 FFE699)
- 폰트: 맑은 고딕 / 헤더 색상: 1F4E78

### Step 5: 캘린더 등록
차기 실적 발표일을 구글 캘린더에 종일 일정으로 등록한다.

**일정 형식**:
- 제목: `[실적] 종목명 (코드) - YYYY NQ`
- 잠정 일정: `[실적/잠정] 종목명 (코드) - YYYY NQ`
- 알림: 1일 전 팝업 (1440분)
- 색상: 반도체(9), AI 전력 인프라(5), 피지컬 AI 로봇(10)

**캘린더 등록 전 사용자 확인 필수**.
**ISO 8601 형식**: `YYYY-MM-DDT00:00:00Z`.

### Step 6: 산출물 요약
엑셀 파일을 present_files로 공유하고 인사이트 요약:
1. 저평가 종목 (현재가 < 보수 적정가)
2. 시장 과열 신호 (컨센서스 목표가 < 현재가)
3. AI 재평가 시 상승 여력 큰 종목
4. 추세 변화 (히스토리가 있는 경우)
5. 다음 분기 체크포인트

## 명령어 간 연계

5개 슬래시 명령어는 독립적이지만 자연스럽게 연계된다:

```
[일반 흐름]
/적정가산정 [신규 종목]   ← 처음 검토 시
    ↓
[포트폴리오 추가 후]
/실적프리뷰 [종목]       ← 차기 실적 전 점검
    ↓ (실적 발표 후)
/실적분석 [종목] [분기]  ← 실적 반영, 히스토리 자동 추가
    ↓ (3~4분기 누적 후)
/논리점검 [종목]         ← 추세 분석으로 매수/보유/매도 판단

[ETF 검토 시]
/ETF분석 [ETF명]         ← 편입 종목 가중평균으로 평가
```

## 데이터의 정직성

추정에는 한계가 있다. 다음 원칙을 산출물과 사용자 응답에 반드시 포함한다:

- "모든 적정가는 가정의 함수입니다. EPS 가정이 10% 틀리면 적정가도 10% 틀립니다."
- "사이클 종목은 정점에서 PER이 가장 낮아 보입니다 (이익 함정)."
- "AI 재평가 시나리오는 단기 목표가가 아닙니다."
- "본 추정은 투자 권유가 아닙니다."
